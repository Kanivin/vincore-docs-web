Inventory Subcontract

The Subcontract area in Inventory Management is used to control subcontract commercial commitments, execution scope, claim review, and invoice recognition.

Navigation​

Open Inventory Management > Subcontract.

The current web UI includes these tabs:

  • Agreement
  • Work Order
  • BOQ / Scope
  • Progress Claim
  • Invoice

The service layer also supports Variation, Retention, and Payment Tracking records, but those tabs are not currently enabled in the web navigation.

End-to-End Flow​

The typical operating flow is:

  1. Maintain reusable BOQ / Scope items.
  2. Create a Subcontract Agreement for the main commercial commitment.
  3. Create a Work Order when execution scope is released.
  4. Capture completed work in Progress Claim.
  5. Convert approved work into Invoice for payable and actual cost recognition.

Common Workflow Rules​

  • Agreement and Work Order statuses: open, submitted, approved, closed, cancelled
  • Progress Claim statuses: open, submitted, approved, rejected, cancelled
  • Invoice statuses: open, submitted, approved, paid, cancelled
  • Records are editable and deletable only while they are in open status.
  • Reopen actions return the document to open so it can be corrected and resubmitted.
  • Each major document keeps workflow history and supports document attachment storage.

Agreement​

Use Agreement to capture the main subcontract commitment before downstream processing starts.

Header fields​

  • Required: Agreement No, Subcontractor, Agreement Date, Job, Cost Code
  • Optional: Start Date, End Date, Phase, Approved By, Retention %, Status, Notes, Document attachment

Line fields​

  • Scope Code
  • Quantity
  • UOM
  • Unit Rate
  • Amount
  • Line Notes

Behavior​

  • Approved agreements project into Committed Cost as the main subcontract commitment.
  • If later work orders are linked to the agreement, the agreement still remains the primary commercial commitment.
  • Printing is available for approved and closed agreements.

Work Order​

Use Work Order to release execution scope either against an existing agreement or as a standalone subcontract instruction.

Header fields​

  • Required: WO No, Subcontractor, WO Date, Job, Cost Code
  • Optional: Agreement reference, Start Date, End Date, Phase, Approved By, Retention %, Status, Notes, Document attachment

Line fields​

  • Scope Code
  • Quantity
  • UOM
  • Unit Rate
  • Amount
  • Line Notes

Behavior​

  • A work order can reference a subcontract agreement.
  • Standalone approved work orders project into Committed Cost.
  • If a work order is linked to an agreement, it acts as execution authorization and does not replace the agreement commitment.
  • Printing is available for approved and closed work orders.

BOQ / Scope​

Use BOQ / Scope to maintain reusable subcontract scope codes that can be reused in agreements, work orders, claims, and invoices.

Fields​

  • Required: Scope Code, Scope Name
  • Optional: Quantity, UOM, Unit Rate, Amount, Status, Notes

Status values​

  • active
  • Hold
  • closed

Behavior​

  • Active scope rows are offered in downstream subcontract documents.
  • When a scope code is selected in a transaction line, the system can prefill quantity, UOM, rate, amount, and description from the scope setup or linked agreement context.

Progress Claim​

Use Progress Claim to capture earned work for review before invoice recognition.

Header fields​

  • Required: Claim No, Subcontractor, Claim Date, Job, Cost Code
  • Optional: Agreement, Work Order, Billing From, Billing To, Phase, Approved By, Status, Notes, Document attachment

Line fields​

  • Scope Code
  • Claim Qty
  • UOM
  • Unit Rate
  • Claim Amount
  • Line Notes

Behavior​

  • Progress claims are operational approval documents.
  • In the current implementation, approved claims do not post actual cost.
  • Actual cost recognition is intentionally deferred until the related invoice is approved.

Invoice​

Use Invoice to convert approved subcontract work into a payable document and recognized actual cost.

Header fields​

  • Required: Invoice No, Subcontractor, Invoice Date, Job, Cost Code
  • Optional: Progress Claim, Agreement, Work Order, Due Date, Phase, Approved By, Status, Notes, Document attachment

Line fields​

  • Scope Code
  • Description
  • Invoice Qty
  • UOM
  • Unit Rate
  • Amount
  • Line Notes

Behavior​

  • Selecting a Progress Claim can auto-fill agreement, work order, subcontractor, budget references, and line items.
  • Selecting an Agreement can auto-select a linked work order when one exists.
  • Approved invoices project into Actual Cost as subcontract actuals.
  • Printing is available for approved and paid invoices.

Budget and Cost Impact​

  • Agreement: contributes to Committed Cost
  • Standalone Work Order: contributes to Committed Cost
  • Progress Claim: approval checkpoint only
  • Invoice: contributes to Actual Cost

This separation helps prevent duplicate cost posting while still preserving commercial and operational approval steps.

Supporting Records Available in Service Layer​

The backend already supports these additional subcontract records:

  • Variation: commercial additions, omissions, or scope changes
  • Retention: held and released retention balances
  • Payment Tracking: planned, submitted, and paid settlements

These are available through service endpoints and can be exposed in the UI when the business flow requires them.
