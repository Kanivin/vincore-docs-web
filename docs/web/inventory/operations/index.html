Inventory Operations

The Inventory & Stores module manages procurement, receiving, stock visibility, warehouse movement, site usage, equipment rental, and direct miscellaneous expense capture.

Navigation​

Open Inventory Management.

The main tabs covered in this guide are:

  • Purchase Order
  • Equipment Rental
  • Rental Usage
  • GRN
  • Stock Management
  • Issue / Return
  • Consumption / Usage
  • Miscellaneous

Operating Flows​

Material procurement and stock flow​

  1. Create a Purchase Order.
  2. Receive vendor delivery through GRN.
  3. Review balances in Stock Management.
  4. Move stock between warehouses or start Warehouse Issue.
  5. Reconcile site usage in Consumption / Usage.

Equipment rental cost flow​

  1. Create Equipment Rental as the committed rental arrangement.
  2. Record Rental Usage against the approved rental header.
  3. Post rental usage when the invoice or billing period should hit actual cost.

Direct expense flow​

  1. Create a Miscellaneous Expense document.
  2. Submit and approve it.
  3. Mark it Paid to project it into Budget Actual Cost.

Common Workflow Rules​

  • Most inventory transaction screens start in open.
  • Documents can generally be edited or deleted only while they are open.
  • Workflow actions move documents through approval stages from the table toolbar.
  • Reopen actions send records back to open so users can correct and resubmit them.
  • Job and Cost Code are core budget references across almost every inventory transaction.

Purchase Order​

Use Purchase Order to create supplier commitments before goods are received in GRN.

Statuses​

  • open
  • submitted
  • approved
  • closed
  • cancelled

Header fields​

  • Required: PO No, Vendor, Order Date, Job, Cost Code
  • Optional: Expected Date, Warehouse, Phase, Approved By, Status, Notes, Document attachment

Line fields​

  • Item
  • Ordered Qty
  • Unit Rate
  • Amount
  • Line Notes

Behavior​

  • Purchase Order is the upstream reference for GRN.
  • GRN uses PO Reference to keep receiving traceable back to the original order.
  • Printing is available for approved and closed purchase orders.

Equipment Rental​

Use Equipment Rental to create the commercial rental commitment for third-party equipment.

Statuses​

  • open
  • submitted
  • approved
  • closed
  • cancelled

Header fields​

  • Required: Rental No, Rental Vendor, Start Date, Job, Cost Code
  • Optional: End Date, Phase, Approved By, Status, Notes, Document attachment

Line fields​

  • Equipment Item
  • Rental Qty
  • Unit Rate
  • Amount
  • Line Notes

Behavior​

  • Approved equipment rentals project into Budget Committed Cost as Equipment Rental.
  • Use Rental Usage later to recognize billed or actual rental cost.

Rental Usage​

Use Rental Usage to post invoice periods or billed usage against an existing equipment rental header.

Statuses​

  • open
  • submitted
  • approved
  • posted
  • cancelled

Header fields​

  • Required: Usage No, Rental Header, Billing From, Status
  • Common optional fields: Invoice No, Billing To, Approved By, Notes, Document attachment
  • Auto-filled from Rental Header: Rental Vendor, Job, Phase, Cost Code

Line fields​

  • Equipment Item
  • Billed Qty
  • Billed Rate
  • Actual Amount
  • Line Notes

Behavior​

  • Rental Header is mandatory.
  • Selecting the rental header can prefill vendor, budget references, and item lines from the rental document.
  • When status becomes posted, the document projects into Budget Actual Cost as Equipment Rental.

GRN​

Use GRN to receive supplier deliveries into warehouse stock.

Statuses​

  • open
  • submitted
  • approved
  • stock-updated

Header fields​

  • Required: GRN No, Delivery Date, Warehouse, Job, Cost Code, PO Reference
  • Optional: Vendor, Phase, Approved By, Status, Notes, Document attachment

Line fields​

  • Item
  • PO Qty
  • Received Qty
  • Rejected Qty
  • Batch No
  • Unit Rate
  • Amount

Behavior​

  • PO Reference must match the selected Job, Phase, and Cost Code context.
  • Selecting a PO reference can auto-fill the vendor from the linked purchase order.
  • GRN moves stock into inventory; after approval the backend transitions the document to stock-updated.
  • Printing is available for approved and stock-updated GRNs.

Stock Management​

Use Stock Management to review live stock position and trigger downstream stock actions.

Main sections​

  • Warehouse Stock Balances: on-hand quantity, average rate, total value, last movement
  • Stock Ledger Entries: transaction history by date, type, reference, direction, quantity, rate, and amount
  • Item Batch Balances: batch-wise quantity, rate, receipt date, and expiry date

Available actions​

  • Move Stock: creates a stock transfer from the selected warehouse balance row
  • Warehouse Issue: launches Issue / Return with the selected stock row prefilled

Transfer fields​

  • Required: Transfer No, Transfer Date, To Warehouse, Transfer Qty, Job, Cost Code
  • Optional: Phase, Requested By, Batch No, Status, Description

Behavior​

  • Transfer can start only from a row with available on-hand quantity.
  • Source and destination warehouses cannot be the same.
  • Transfer quantity cannot exceed available stock.
  • Job filtering is available when stock rows contain project references.

Issue / Return​

Use Issue / Return for warehouse stock-out and stock-in transactions.

Movement types​

  • Issue
  • Return

Workflow statuses​

  • open
  • submitted
  • approved

The backend finalizes approved GRN-style stock movements as stock-updated, so approved Issue / Return transactions are used to update warehouse stock.

Header fields​

  • Required: Movement Type, Document No, Movement Date, Warehouse, Status
  • Required for Issue: Job, Cost Code
  • Optional: Phase, Employee, Subcontractor, Issue Reference, Notes

Line fields​

  • Item
  • Batch No
  • Quantity
  • Unit Rate
  • Amount
  • Issue Reference for return lines

Behavior​

  • Issue moves stock out of the warehouse.
  • Return moves stock back in and should reference the original issue.
  • Quick Issue can be launched directly from Stock Management.
  • Return requires an Issue Reference either in the header or on each line.

Consumption / Usage​

Use Consumption / Usage to record site usage against already issued material.

Statuses​

  • open
  • submitted
  • approved

Header fields​

  • Required: Usage No, Job, Cost Code, Status
  • Optional: Phase, Notes

Line fields​

  • Issue Reference
  • Item
  • Available Qty
  • Consumed Qty
  • Unit Rate
  • Amount
  • Consumed By
  • Consumption Date

Behavior​

  • Consumption can only be recorded from issued material with remaining balance.
  • Available issue references are filtered by selected Job, Phase, and Cost Code.
  • Warehouse stock is not reduced here again; stock reduction already happened at Issue / Return.
  • Consumed quantity cannot exceed the available issued balance for that item and issue reference.

Miscellaneous​

Use Miscellaneous Expense for direct site and petty expenses outside stock issues.

Statuses​

  • open
  • submitted
  • approved
  • paid

Header fields​

  • Required: Expense No, Expense Date, Job, Cost Code, Miscellaneous Type, Payment Type, Status
  • Phase is required when the environment requires phase tracking.
  • Optional: Vendor / Payee, Reference No, Approved By, Notes, Document attachment

Line fields​

  • Description
  • Qty
  • UOM
  • Unit Rate
  • Amount

Behavior​

  • Intended for fuel, food, local transport, courier, printing, internet recharge, temporary labour, repairs, and similar direct expenses.
  • Budget Actual Cost recognizes miscellaneous expense amounts only after the document reaches paid.

Cost and Stock Impact Summary​

  • Purchase Order: procurement commitment, upstream reference for GRN
  • Equipment Rental: contributes to Budget Committed Cost
  • Rental Usage: contributes to Budget Actual Cost when posted
  • GRN: increases warehouse stock
  • Stock Management: inquiry and action screen for balances, ledger, and transfers
  • Issue / Return: decreases or restores warehouse stock
  • Consumption / Usage: consumes issued balance without reducing warehouse stock again
  • Miscellaneous Expense: contributes to Budget Actual Cost when paid
